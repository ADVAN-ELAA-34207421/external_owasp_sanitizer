OWASP Java HTML Sanitizer Change Log

  1. Cleaned up raw-text tag handling. XMP, LISTING, PLAINTEXT now handled by substitution in the renderer and changed NOSCRIPT and friends so they are treated consistently when elided as when present in output. Added workaround for IE8 innerHTML wierdness.
  2. Prevent DoS of browsers via extremely deeply nested tags. In sanitized CSS, allow CSS property background-color andfont-sizes specified in px.
  3. Added convenient pre-packaged policies in Sanitizers. Fixed bug in how warnings are reported via the badHtml Handler.
  4. Better handling of supplementary codepoints to avoid UTF-16/UCS-2 confusion in browsers.
  5. Added new HTML5 URL attributes to list used to safeguard URL attributes in HtmlPolicyBuilder.
  6. Changed HtmlSanitizer.sanitize to allow null as a valid value for the HTML snippet.
